## 1. Design System

```

```

```
2. Redirect to Google OAuth consent screen
3. Google redirects back to /auth/callback
4. Backend creates session, redirects to /review-queue
5. Frontend fetches /auth/me to get user info
```

### 7.4 State Flow: Editing a Block

```
1. User types in MarkdownEditor
2. MarkdownEditor calls onChange(newValue)
3. BlockEditor updates workingBlocks map
4. BlockEditor validates content
5. onValidationChange updates validationErrors
6. DiffPanel recalculates diff
7. DecisionControls updates hasUnsavedChanges
```

### 7.5 State Flow: Submitting Decision

```
1. User clicks Approve
2. ApproveModal opens
3. User confirms
4. POST /review-tasks/{id}/decision with blocks_working
5. On success:
   - Update task.status to APPROVED
   - Set all editors to read-only
   - Show success message
6. On error:
   - Show error in modal
   - Keep modal open
```
